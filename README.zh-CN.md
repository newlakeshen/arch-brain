# arch-brain

[English](README.md)

一个适用于 Claude Code 和 Codex CLI 的 skill，充当观点鲜明的资深软件架构师顾问，用于架构评审、重构分析和架构方案设计，支持证据化评分、风险情景推演和结构化报告输出。

## 功能特性

- **项目架构了解** — 自动识别技术栈、架构风格、项目规模和团队特征
- **架构模式库** — 识别架构模式（分层、微服务、六边形、CQRS、事件驱动等）并检测反模式
- **证据化评分** — 基于具体评分标准的量化评估，附判定标签（✅/⚠️/🚨）
- **风险情景分析** — What-If 推演（10x 流量、服务故障、技术债累积、团队变动）
- **顾问风格** — 敢于说"必须"和"不应该"，每个观点附具体理由
- **架构可视化** — Mermaid 图（≤15 节点）展示模块关系、部署拓扑和流程
- **ADR 生成** — 在报告附录中生成架构决策记录

## 工作流程

四阶段交互式流程，每阶段结束有用户确认节点：

```
阶段 1: 探索与理解 → 阶段 2: 架构对话 → 阶段 3: 评估分析 → 阶段 4: 输出报告
```

1. **探索与理解** — 项目架构了解（优先读取 CLAUDE.md/README.md，再扫描技术元数据）或对话引导收集信息
2. **架构对话** — 目标、约束、痛点；反模式检测；上下文驱动维度推荐
3. **评估分析** — 证据化评分（具体评分标准）、判定标签、风险情景分析、顾问风格输出
4. **输出报告** — 含项目画像、核心观点、Mermaid 图、风险情景和 ADR 附录的结构化报告

## 安装

```bash
git clone https://github.com/newlakeshen/arch-brain.git
cd arch-brain
```

**Claude Code**（`~/.claude/skills/arch-brain/`）：

```bash
./install_to_claude.sh
./install_to_claude.sh uninstall  # 卸载
```

**Codex CLI**（`~/.codex/skills/arch-brain/`）：

```bash
./install_to_codex.sh
./install_to_codex.sh uninstall  # 卸载
```

## 使用方式

在 Claude Code 中通过 `/arch-brain` 调用：

```bash
# 分析代码库（触发架构了解）
/arch-brain ./src

# 分析架构文档
/arch-brain docs/architecture.md

# 带意图描述的分析
/arch-brain ./src "想拆成微服务"

# 对话引导模式（无参数，跳过架构了解）
/arch-brain
```

## 评估维度

### 核心维度（1-10 评分 + 判定标签）

| 维度 | 评估要点 | 判定 |
|------|---------|------|
| 性能 | 响应时间(P99)、吞吐量、资源利用率 | ✅ 7+ / ⚠️ 4-6 / 🚨 1-3 |
| 安全性 | 攻击面、认证授权、数据保护、合规性 | ✅ 7+ / ⚠️ 4-6 / 🚨 1-3 |
| 可扩展性 | 水平/垂直扩展能力、弹性、无状态设计 | ✅ 7+ / ⚠️ 4-6 / 🚨 1-3 |
| 可维护性 | 模块耦合度、变更成本、新人上手时间 | ✅ 7+ / ⚠️ 4-6 / 🚨 1-3 |
| 投入成本 | 开发人力、迁移成本、基础设施成本 | ✅ 7+ / ⚠️ 4-6 / 🚨 1-3 |
| 收益分析 | 业务价值、技术债务减少、效率提升 | ✅ 7+ / ⚠️ 4-6 / 🚨 1-3 |

### 辅助维度（高/中/低 风险评级）

可测试性、可观测性、团队认知负荷、技术债务、迁移风险、数据一致性、可用性/容灾、部署复杂度

> 维度基于项目特征自动推荐（如：高流量 → 性能 + 可扩展性 + 安全性），经用户确认后进行评估。

## 报告输出

报告保存至 `docs/arch-brain/reports/YYYY-MM-DD-<topic>.md`，包含：

- **项目画像** — 架构了解阶段自动生成
- **架构师核心观点** — 3-5 条关键发现，使用强措辞
- **现状分析** — 含 Mermaid 架构图
- **核心维度评分** — 证据化评分 + 判定标签（✅/⚠️/🚨）
- **辅助维度评级** — 风险等级评估
- **风险情景分析** — 2-3 个 What-If 情景及应对方案
- **架构建议方案** — 至少 2 个方案，可选对比图
- **实施路线图** — 短期/中期/长期 + 风险缓解
- **ADR 附录** — 评审中识别的架构决策记录
