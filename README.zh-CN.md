# arch-brain

[English](README.md)

一个充当资深软件架构师角色的 Claude Code skill，用于架构评审、重构分析和架构方案设计，输出包含多维度量化评估的结构化架构评估报告。

## 功能特性

- **架构评审** — 对现有系统架构进行量化评分评估
- **重构分析** — 诊断痛点，提出改进路径
- **方案设计** — 对比架构方案，分析权衡取舍
- **结构化报告** — 生成多维度评估报告，包含可操作的改进建议

## 工作流程

Skill 通过四阶段交互式流程引导用户：

```
阶段 1: 探索与理解 → 阶段 2: 架构对话 → 阶段 3: 评估分析 → 阶段 4: 输出报告
```

1. **探索与理解** — 分析代码库结构、依赖、技术栈，或通过对话收集项目背景
2. **架构对话** — 一问一答式深入理解目标、约束和痛点
3. **评估分析** — 逐维度进行有据可查的评估（核心维度 1-10 分，辅助维度 高/中/低）
4. **输出报告** — 生成结构化评估文档，保存至 `docs/arch-brain/reports/`

## 安装

```bash
mkdir -p ~/.claude/skills/arch-brain
cp SKILL.md report-template.md ~/.claude/skills/arch-brain/
```

## 使用方式

在 Claude Code 中通过 `/arch-brain` 调用：

```bash
# 分析代码库
/arch-brain ./src

# 分析架构文档
/arch-brain docs/architecture.md

# 带意图描述的分析
/arch-brain ./src "想拆成微服务"

# 对话引导模式（无参数）
/arch-brain
```

## 评估维度

### 核心维度（1-10 评分）

| 维度 | 评估要点 |
|------|---------|
| 性能 | 响应时间、吞吐量、资源利用率、瓶颈分析 |
| 安全性 | 攻击面、认证授权、数据保护、合规性 |
| 可扩展性 | 水平/垂直扩展能力、弹性、无状态设计 |
| 可维护性 | 代码可读性、模块耦合度、变更成本 |
| 投入成本 | 开发人力、迁移成本、学习曲线、基础设施成本 |
| 收益分析 | 业务价值、技术债务减少、效率提升 |

### 辅助维度（高/中/低 风险评级）

可测试性、可观测性、团队认知负荷、技术债务、迁移风险、数据一致性、可用性/容灾、部署复杂度

> 维度根据项目特征动态选择，经用户确认后进行评估。

## 报告输出

报告保存至 `docs/arch-brain/reports/YYYY-MM-DD-<topic>.md`，包含：

- 执行摘要
- 现状分析
- 核心维度评分及详细分析
- 辅助维度风险评级
- 架构建议方案（至少 2 个方案）
- 实施路线图（短期/中期/长期）
- 决策建议和行动项
